# Repo Structure Instructions

This repo should become a deep style-and-structure archive for lace patterns.

It should support:

- visual style definition
- motif dictionaries
- lace family documentation
- prompt generation
- shader and generative art translation
- textile / apparel / print production
- cross-repo mixing with crochet, weaving, surface FX, dither, op art, fractals, and glitch
- anti-drift guidance

## Recommended folder logic

- `00_foundation/` defines what lace is and how to think about it.
- `01_lace_families/` documents named lace families and methods.
- `02_motif_grammar/` documents flowers, vines, grids, scallops, medallions, bars, and grounds.
- `03_thread_and_void_logic/` treats lace as graph, mesh, negative space, and structural topology.
- `04_stitches_and_construction/` documents stitches and construction units.
- `05_visual_language/` turns lace into visual traits for image generation.
- `06_prompt_system/` contains prompt formulas and reusable modules.
- `07_shader_and_genart/` contains procedural translations, shader logic, and code.
- `08_examples/` contains working examples.
- `09_drift_control/` prevents generic lace slop.
- `10_cross_repo_bridges/` connects lace to other repos.
- `11_production_translation/` handles screenprint, laser-cut, embroidery, and vector use.
- `12_structured_data/` contains JSON and CSV indexes.
- `13_reference/` contains cheatsheets and research backlog.

## File expansion rule

Every lace-family file should answer:

1. What is it?
2. How is it made?
3. What does it look like?
4. What are its signature motifs?
5. What are its failure modes?
6. How do you prompt it?
7. How can it become shader / SVG / procedural code?
